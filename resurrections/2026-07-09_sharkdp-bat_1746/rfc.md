# RFC: Support for different themes based on Terminal color scheme

# Summary

This RFC proposes the enhancement of the `bat` command-line tool to automatically detect the terminal's color scheme and apply a corresponding theme. The goal is to improve the user experience by ensuring that `bat` output looks good in both light and dark terminal themes.

## Motivation

The motivation behind this proposal is to make `bat` more user-friendly and adaptable to different terminal settings. Currently, `bat` applies a fixed theme that may not look optimal in all terminal color schemes. By automatically detecting the terminal's color scheme and switching between light and dark themes, `bat` can provide a better reading experience for users.

## Detailed Design

To implement this feature, we will utilize a Rust library like `termenv` to detect the terminal's color scheme. Here's a high-level overview of the design:

1. **Library Selection**: We will use `termenv` for detecting the terminal's color scheme. This library provides a reliable and cross-platform way to achieve this.

2. **Configuration Options**: We will introduce two new configuration options:
   - `--theme-dark` (or `$BAT_THEME_DARK` environment variable) to specify the theme for dark terminal color schemes.
   - `--theme-light` (or `$BAT_THEME_LIGHT` environment variable) to specify the theme for light terminal color schemes.

3. **Theme Detection**: On startup, `bat` will use `termenv` to detect the terminal's color scheme.

4. **Theme Application**: Based on the detected color scheme, `bat` will apply the corresponding theme. If the user has specified a theme for their preferred color scheme, `bat` will use that; otherwise, it will default to a predefined theme.

## Drawbacks

1. **Additional Dependency**: Introducing `termenv` as a dependency may increase the overall size of the `bat` package.

2. **Complexity**: Adding a new feature always introduces some complexity, which could potentially lead to bugs or maintenance overhead.

## Alternatives

1. **Manual Theme Switching**: Users could manually switch between themes using configuration files or command-line arguments. However, this approach does not align with the goal of automating the process based on the terminal's color scheme.

2. **Using Existing Libraries**: Other libraries like `termbg` or `rust-dark-light` could be considered. However, their limitations or bugs make `termenv` a more suitable choice at this time.
