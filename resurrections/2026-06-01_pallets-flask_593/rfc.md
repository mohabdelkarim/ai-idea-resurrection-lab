# RFC: Nestable blueprints

## 1. Summary

This RFC proposes the implementation of nestable blueprints in Flask, allowing for more modular and extensible application design. The goal is to enable registering sub-blueprints within a parent blueprint, which can then be registered with the application.

The current implementation of blueprints in Flask does not support nesting, which can lead to complex and hard-to-maintain applications. As the Flask ecosystem and applications continue to grow, there is a need for more modular and reusable code. Nestable blueprints will enable developers to create more organized and scalable applications.

The main drawbacks of this proposal are:

*   Implementing a robust conflict resolution mechanism.

*   Optimizing the registration and resolution process for sub-blueprints.
