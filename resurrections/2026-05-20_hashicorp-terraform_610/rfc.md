However, the output of these provisioners is not currently captured or exposed in a way that can be used by other parts of the Terraform configuration. This limitation makes it difficult to integrate provisioner outputs with other Terraform features, such as resource dependencies, data sources, and module outputs.




The motivation behind this feature is to unlock the potential of provisioners by making their output accessible. This would enable a wide range of use cases, such as:

* Using provisioner output to configure dependent resources
* Integrating provisioner output with data sources and module outputs
* Enhancing the automation and dynamic configuration capabilities of Terraform
