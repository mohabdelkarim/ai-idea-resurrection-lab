# RFC: cmd/compile: implement powerpc 32-bit backend (ppc32)

## Summary

This RFC proposes the implementation of a PowerPC 32-bit (ppc32) backend for the Go compiler. The goal is to enable Go programs to compile and run on PowerPC 32-bit architectures, expanding the reach of the Go language.

## Motivation

The PowerPC 32-bit architecture, although less common than other architectures like amd64 or arm64, still has relevance in certain embedded systems and legacy environments. Supporting ppc32 would allow Go to be used in these contexts, enhancing its versatility and adoption. The implementation of this backend would also align with Go's goal of being a viable choice for a wide range of platforms.

## Detailed Design

The implementation of the ppc32 backend would build upon the existing infrastructure of the Go compiler, specifically leveraging the Static Single Assignment (SSA) representation and the architecture-agnostic components of the 'gc' compiler. 

Key steps would include:
- Defining the ppc32 architecture profile, including its register layout and addressing modes.
- Implementing the SSA-to-ppc32 instruction selection and lowering logic.
- Handling ppc32-specific instructions, such as those for loading and storing data, and for performing arithmetic and logical operations.
- Integrating the ppc32 backend with the existing Go toolchain, ensuring compatibility with other components like the assembler and linker.

## Drawbacks

- **Complexity**: Implementing a new backend requires a deep understanding of both the Go compiler's internals and the target architecture. This could introduce complexity and potential for bugs.
- **Maintenance**: The ppc32 backend would need to be maintained over time, which could be challenging given the relatively low usage of the architecture.
- **Resource Usage**: The addition of a new backend might increase the resource usage (e.g., binary size, compile time) of the Go toolchain.

## Alternatives

- **LLVM-based Approach**: Instead of implementing a custom ppc32 backend, consider using the LLVM compiler infrastructure, which already supports PowerPC 32-bit. However, integrating LLVM might add complexity and affect the Go toolchain's overall design and performance.
- **Prioritizing Other Architectures**: Focus on architectures with higher usage and demand, potentially deferring support for ppc32 until later.
