# RFC: Please expose current user and group ID as new variables for use inside docker-compose.yml

Summary

--------

Expose the current user and group ID as variables in docker-compose.yml to simplify container user management.



Motivation

----------

Managing user and group IDs in Docker containers can be cumbersome. Currently, users have to manually retrieve their IDs using commands like `id -u` and `id -g`, and then hardcode them in their docker-compose.yml files. This approach is not only inconvenient but also prone to errors. By exposing the current user and group ID as variables, users can easily reference them in their docker-compose.yml files, making it easier to manage container users.



Detailed Design

---------------

We propose adding two new variables, COMPOSE_USER_ID and COMPOSE_GROUP_ID, to docker-compose.yml. These variables will be populated with the host system's user and group IDs, respectively. The implementation will involve modifying the docker-compose.yml parser to recognize these variables and replace them with the actual IDs.



To ensure security, the implementation will follow these guidelines:

- The variables will only be expanded when necessary, i.e., when used in a docker-compose.yml file.

- The variables will be populated using the host system's user and group IDs, which will be retrieved using the `os` package in Go.

- The implementation will be designed to work with Docker Compose v2.



Drawbacks

----------

The main drawback of this proposal is the potential security risk associated with exposing user and group IDs. However, this risk can be mitigated by ensuring that the variables are only expanded when necessary and that they are not stored or transmitted in plaintext.



Alternatives

------------

One alternative to this proposal is to use an external script to populate the user and group IDs in the docker-compose.yml file. However, this approach is more complex and prone to errors than the proposed solution.
