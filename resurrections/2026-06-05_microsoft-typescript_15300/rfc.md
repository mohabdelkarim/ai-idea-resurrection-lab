# RFC: Index signature is missing in type (only on interfaces, not on type alias)

Summary

--------

This RFC proposes to enhance the TypeScript type system by allowing index signatures on interfaces. This change aims to improve type compatibility between interfaces and type aliases, making interfaces more versatile and aligned with type aliases.


Motivation

-----------

Currently, TypeScript allows index signatures on type aliases but not on interfaces. This inconsistency limits the usability of interfaces in scenarios where type aliases are more flexible. For example, when defining a type that represents a collection of properties, interfaces are more commonly used due to their extendability and implementability. However, the lack of index signatures on interfaces forces developers to choose type aliases, which might not offer the same level of abstraction or tooling support.


The primary motivation is to reconcile the behavior between interfaces and type aliases regarding index signatures. This enhancement would make interfaces more competitive and flexible, allowing developers to use them in a broader range of scenarios without sacrificing type safety.
