----------

Currently, `ansible-test` requires that tests be run from within a collection root or an Ansible source directory. This limitation can be restrictive for users who need to run tests from different directories, such as project-specific test directories or CI/CD environments. By lifting this restriction, users will have more freedom in their workflow and project organization.

The motivation behind this change is driven by user feedback and the evolving needs of the Ansible community. As projects grow in complexity and teams adopt more sophisticated CI/CD pipelines, the ability to run tests from arbitrary directories becomes increasingly important.
