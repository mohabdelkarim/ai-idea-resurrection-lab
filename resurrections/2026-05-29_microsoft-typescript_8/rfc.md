# RFC: Suggestion: minification

## 1. Summary

This RFC proposes that the TypeScript compiler (tsc) support emitting minified JavaScript code natively. The goal is to provide developers with a convenient, out-of-the-box way to minify their JavaScript code without having to rely on external tools.

## 2. Motivation

Currently, TypeScript developers rely on external tools like UglifyJS, Closure Compiler, or Terser to minify their JavaScript code. While these tools are effective, they add an extra step to the development workflow. By integrating minification directly into the TypeScript compiler, we can simplify the process and make it more efficient for developers to produce minified code.

The motivation behind this proposal is to:

* Improve developer productivity by reducing the number of tools they need to manage
* Provide a consistent, first-class minification experience for TypeScript users
* Take advantage of advancements in JavaScript bundlers and minification tools

## 3. Detailed Design

To implement native minification support in TypeScript, we propose the following:

* Add a new compiler option, `--minify` (or `minify: true` in tsconfig.json), which enables minification
* Utilize a minification library like Terser under the hood
* Create a new class, `Minifier`, which will implement the minification logic
* Update the `tsc` compiler to emit minified JavaScript code when the `--minify` option is enabled

The `Minifier` class will be responsible for:

* Analyzing the generated JavaScript code
* Applying minification techniques, such as:
	+ Removing unnecessary whitespace and comments
	+ Renaming variables and functions to shorter names
	+ Eliminating dead code
* Emitting the minified code

## 4. Drawbacks

While the benefits of native minification support are clear, there are some potential drawbacks to consider:

* Increased complexity in the TypeScript compiler
* Potential performance impact on the compilation process
* The need to maintain and update the minification library (e.g., Terser) to ensure compatibility with the latest JavaScript features

## 5. Alternatives

Before proposing this RFC, we considered the following alternatives:

* Continue to rely on external minification tools, which would require no changes to the TypeScript compiler
* Integrate with existing bundlers like Webpack or Rollup, which often include minification capabilities
* Use a different minification library or implement a custom minification solution

However, we believe that native minification support in TypeScript offers the most convenient and streamlined experience for developers.

## 6. Unresolved Questions

While this RFC outlines a general approach to implementing native minification support in TypeScript, there are still some unresolved questions:

* How will the `--minify` option interact with existing compiler options, such as `--target` and `--module`?
* Will the minified code be compatible with all browsers and environments?
* How will we ensure that the minification process does not introduce bugs or issues with the generated code?

These questions will need to be addressed as we move forward with implementing this feature.
