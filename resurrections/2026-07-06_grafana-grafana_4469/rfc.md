# RFC: [Feature Request] Option to align/match Y-axis width 

## Summary

This RFC proposes the addition of a new feature to Grafana's graph panels: the ability to align Y-axis widths across multiple panels. This feature will enable users to easily compare data across different graphs, making it easier to identify trends and patterns.

## Motivation

The current implementation of graph panels in Grafana does not provide a straightforward way to compare data across multiple graphs. When graphs have different Y-axis scales, it can be challenging to compare data points across graphs, especially when the graphs are rendered side-by-side. By adding an option to align Y-axis widths, users will be able to easily compare data across different graphs, making it easier to gain insights and make data-driven decisions.

## Detailed Design

To implement this feature, we propose the following:

* Introduce a new configuration option for graph panels, `alignYAxisWidths`, which will be a boolean flag that defaults to `false`.
* When `alignYAxisWidths` is enabled, the graph panel will calculate the maximum Y-axis width across all panels in a row.
* The graph panel will then adjust its own Y-axis width to match the maximum width calculated in the previous step.
* To ensure a consistent look and feel, we will utilize Grafana's existing theme and styling system.

## Drawbacks

The main drawback of this feature is that it may introduce additional computational overhead, as the graph panel will need to calculate the maximum Y-axis width across all panels in a row. However, this overhead is expected to be minimal and will not significantly impact the user experience.

## Alternatives

One alternative to this approach is to use a fixed Y-axis width for all graph panels. However, this approach has several drawbacks:

* It may not work well for graphs with varying Y-axis scales.
* It may not be suitable for graphs with different units or labels.

Another alternative is to use a more advanced layout algorithm that takes into account the Y-axis widths of adjacent panels. However, this approach is more complex and may require significant changes to Grafana's rendering engine.
