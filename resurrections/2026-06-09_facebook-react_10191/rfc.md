# RFC: Consider re-licensing to AL v2.0, as RocksDB has just done

## 1. Summary

This RFC proposes re-licensing the React repository from its current Facebook BSD+Patents License to the Apache License v2.0 (AL v2.0). This change aims to enhance compatibility with other open-source projects and align with modern open-source licensing practices.

## 2. Motivation

The React repository currently uses the Facebook BSD+Patents License. However, this license may not be as widely compatible or as permissive as modern projects require. Recently, RocksDB re-licensed to AL v2.0, setting a precedent for improved compatibility. The Apache License v2.0 is a widely adopted, permissive license that facilitates collaboration and reuse across the open-source ecosystem.

## 3. Detailed Design

To re-license React under AL v2.0, the following steps are proposed:

1. **License Assessment**: Conduct a thorough review of the current licensing landscape of the React repository, including all dependencies and contributions.
2. **Contributor Coordination**: Reach out to past and present contributors to inform them of the proposed license change and gather their consent if necessary.
3. **License Scanning**: Utilize tools like license scanners to identify any components that may require re-licensing or removal.
4. **Code Updates**: Update license headers and ensure all new code is licensed under AL v2.0. Re-license or remove code that cannot be re-licensed.
5. **Dependency Compatibility**: Verify that the re-licensed React is compatible with its existing dependencies and does not introduce any licensing conflicts.
6. **Documentation and Communication**: Update all relevant documentation to reflect the new licensing terms and communicate the change to the community and stakeholders.

## 4. Drawbacks

1. **Complexity and Effort**: The process of re-licensing a large and complex project like React requires significant effort and resources.
2. **Contributor Consent**: Obtaining consent from all contributors may be challenging, especially for those who contributed under the original licensing terms.
3. **Potential for Forks**: Some contributors or users might prefer to stick with the existing license, potentially leading to forks of the project.

## 5. Alternatives

1. **Status Quo**: Maintain the current licensing scheme. However, this might limit React's compatibility and adoption in projects that prefer more permissive licenses.
2. **Other Licenses**: Consider alternative permissive licenses. However, AL v2.0 is widely recognized and adopted, making it a preferable choice for compatibility.
