# RFC: Option to disable all Cloud and remote Search features

## 1. Summary

This RFC proposes the implementation of a configuration option to disable all remote and cloud features in Ollama, enhancing user privacy and control over data.

## 2. Motivation

The motivation behind this RFC is to provide users with an explicit option to opt-out of remote and cloud features in Ollama. This is driven by growing concerns over data privacy and the need for users to have control over their data. The existing safeguards that prevent remote features from activating without a login or token do not fully address the need for a 'local-only' mode. With modern tooling and configuration management capabilities, implementing such a feature without significant overhead is feasible.

## 3. Detailed Design

The proposed feature will be implemented by introducing a configuration flag that toggles remote/cloud features on or off. This flag, named 'local_only_mode', will be a boolean configuration option that defaults to false. When set to true, all remote features will be disabled.

The 'local_only_mode' flag will influence the initialization of remote connections and model loading mechanisms. 

- When 'local_only_mode' is false (default), Ollama will function as it does today, allowing remote features.
- When 'local_only_mode' is true, Ollama will disable all remote features, including but not limited to:
  - Remote model fetching
  - Cloud-based search
  - Any form of data exchange with remote servers

## 4. Drawbacks

The primary drawback of implementing this feature is the potential loss of functionality for users who rely on remote and cloud features. This could include users who depend on access to a wide range of models not available locally or those who use cloud-based services for data analysis and collaboration.

## 5. Alternatives

An alternative approach could involve more granular controls, allowing users to selectively disable certain remote features while keeping others enabled. However, this approach could add complexity to the configuration and user interface, potentially leading to user confusion.

- What are the implications for users who are currently using remote features? Will there be a migration path or guidance on what to expect when 'local_only_mode' is enabled?
- Are there any potential security implications of disabling remote features that need to be considered?
