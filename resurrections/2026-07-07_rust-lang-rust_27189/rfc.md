# RFC: Add `rustc --fatal` which stops at the first build error

# Summary

This RFC proposes the addition of a new command-line flag to the Rust compiler, `rustc --fatal`, which causes the compiler to exit immediately after encountering its first build error. This flag aims to improve the usability of the Rust compiler by allowing users to quickly identify and address errors without having to sift through a large number of errors.




# Motivation

The Rust compiler is designed to report multiple errors in a single run, which can be helpful for identifying and fixing a range of issues in a codebase. However, this behavior can sometimes be overwhelming, especially for beginners or when working on large projects. By providing a `--fatal` flag, users can opt to stop the compilation process as soon as the first error is encountered, making it easier to diagnose and fix issues.

The `--fatal` flag would be implemented by modifying the compiler's error handling logic to exit immediately after encountering the first fatal error. This involves:

*   Adding a new flag to the `rustc` CLI parser
*   Modifying the error diagnostic system to handle the new flag
*   Ensuring that the compiler exits cleanly after reporting the error

The implementation would likely involve changes to the following components:

*   `rustc` command-line argument parser: to recognize and parse the `--fatal` flag
*   Error diagnostic system: to handle the new flag and exit immediately after reporting the first fatal error
*   Compiler exit logic: to ensure that the compiler exits cleanly after reporting the error




# Drawbacks

The main drawback of this proposal is that it changes the default behavior of the Rust compiler. Currently, the compiler reports all errors it encounters during a single run. By adding a `--fatal` flag, users may need to adjust their workflow to accommodate the new behavior.

Additionally, there may be concerns about backwards compatibility, particularly if existing tooling or scripts rely on the compiler's current behavior.




# Alternatives

An alternative approach could be to implement a `--error-limit` flag, which would allow users to specify a maximum number of errors to report before exiting. However, this approach is more complex and may not provide the same level of usability benefits as the `--fatal` flag.

Another alternative is to modify the compiler's error reporting behavior to prioritize errors over warnings, but this would not provide the same level of control as the `--fatal` flag.
