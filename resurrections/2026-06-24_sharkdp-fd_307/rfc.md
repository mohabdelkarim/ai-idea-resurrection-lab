# RFC: Finding by owner username/group/uid/gid

# Summary

This RFC proposes the addition of a new feature to the `fd` command-line tool, allowing users to search for files by owner username, group, uid, or gid. The feature will be implemented through a new flag, `--owner`, which accepts a string in the format of `[(uid|username)][:(gid|groupname)]`.




# Motivation

The `fd` tool is a powerful file search utility that currently supports various filtering options, such as file name patterns, file type, and directory traversal. However, it lacks the ability to filter files based on their owner or group information. This feature will enhance the usability of `fd` for system administrators and users who need to manage files based on ownership.




# Detailed Design

The `--owner` flag will accept a string argument in the format of `[(uid|username)][:(gid|groupname)]`. The string can contain either a uid, username, gid, or groupname, or a combination of them separated by a colon. 

- If only a uid or username is provided, `fd` will search for files owned by that user.
- If only a gid or groupname is provided, `fd` will search for files owned by a group with the matching gid or name.
- If both are provided, `fd` will search for files owned by the specified user and group.

The implementation will utilize the `users` and `gid` crates to parse and validate the input. The `--owner` flag will be mutually exclusive with other flags that affect file filtering.

Example usage:

- `fd --owner username /path/to/search`
- `fd --owner 1000 /path/to/search`
- `fd --owner username:groupname /path/to/search`
- `fd --owner :gid /path/to/search`




# Drawbacks

- The feature adds complexity to the `fd` tool, potentially affecting performance.
- The usage of the `--owner` flag might be less intuitive for users not familiar with Unix user and group management.
- There might be platform-specific issues with retrieving user and group information.




# Alternatives

- An alternative approach could be to use external tools, such as `find`, which already supports filtering by owner and group. However, integrating this feature into `fd` will provide a more streamlined and consistent user experience.
