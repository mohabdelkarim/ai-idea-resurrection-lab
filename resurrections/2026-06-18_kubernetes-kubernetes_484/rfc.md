# RFC: Service scale-to-zero (fka "Socket activation")

# Summary

This RFC proposes implementing socket activation, also known as scale-to-zero, in Kubernetes. The goal is to enable pods to scale down to zero replicas when not in use, reducing memory usage and improving resource efficiency. # Motivation

The motivation behind this feature is to optimize resource utilization in Kubernetes clusters. Many applications, especially those built using serverless frameworks, experience periods of inactivity or low traffic. During these periods, the resources allocated to these applications are underutilized, leading to waste and increased costs. By scaling pods to zero when not in use, we can reduce memory usage and improve resource efficiency.
