# RFC: Please, oh please, ask, or at least warn, about overwriting ~/.zshrc on installation

## 1. Summary

Implement a warning system to prevent accidental overwriting of ~/.zshrc during Oh My Zsh installation.

The proposed solution involves modifying the Oh My Zsh installer to check if the ~/.zshrc file already exists and, if so, prompt the user for confirmation before overwriting it. This will be achieved using a Python library like `questionary` to create a text-based interface.

## 2. Motivation

The current Oh My Zsh installer does not provide a warning or prompt when overwriting the ~/.zshrc file. This can lead to accidental loss of user customizations and configurations. The goal of this RFC is to implement a warning system that informs users about the potential overwrite and provides them with the option to cancel or proceed with caution.

## 3. Detailed Design

The proposed solution involves the following steps:

1. Check if the ~/.zshrc file already exists.
2. If the file exists, use `questionary` to prompt the user with a warning message indicating that the file will be overwritten.
3. Provide the user with options to:
	* Proceed with the installation (overwrite ~/.zshrc).
	* Cancel the installation.
	* Choose an alternative installation option (e.g., append to ~/.zshrc instead of overwriting).
4. Based on the user's response, either proceed with the installation, exit the installer, or provide alternative installation options.

The warning message should clearly indicate the potential risks of overwriting the ~/.zshrc file and provide guidance on how to backup user customizations.

## 4. Drawbacks

The main drawback of this solution is that it may cause minor disruptions to the installation process. However, the benefits of preventing accidental overwrites and providing users with more control over their configuration files outweigh the costs.

## 5. Alternatives

Alternative solutions include:

1. Appending to ~/.zshrc instead of overwriting: This approach would require significant changes to the installer and may not be desirable for users who want a clean ~/.zshrc file.
2. Providing a pre-installation checklist: This approach would require users to manually check their ~/.zshrc file before installing Oh My Zsh.
