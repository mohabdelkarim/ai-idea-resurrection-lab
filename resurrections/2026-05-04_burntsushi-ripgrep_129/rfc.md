# RFC: maximum line length

## Summary
   This RFC proposes the addition of a maximum line length feature to the ripgrep library. The feature would allow users to specify a maximum line length and truncate lines that exceed this length.

   ## Motivation
   The motivation behind this feature is to improve the usability of ripgrep when searching through large files with very long lines. Currently, these lines can take up the entire screen and make it difficult to read the search results.

   ## Detailed Design
   The feature would be implemented by adding a new option to the ripgrep CLI that allows users to specify a maximum line length. The rendering logic would be updated to use a custom truncation algorithm that preserves the matched text.

   ## Drawbacks
   One potential drawback of this feature is that it may not work well with certain types of files, such as CSV or JSON files, where the lines are not necessarily text-based.

   ## Alternatives
   One alternative to this feature is to use a different search tool that has built-in support for truncating lines. However, this may not be desirable for users who are already invested in the ripgrep ecosystem.

   ## Unresolved Questions
   One unresolved question is how to handle cases where the matched text is near the end of a very long line. Should the truncation algorithm prioritize preserving the matched text or the surrounding context?
