# RFC: promises

## 1. Summary

This RFC proposes enhancing Express.js to support promises for async-friendly middleware. The goal is to allow developers to write asynchronous middleware functions that return promises, which can be resolved by the Express.js core. This enhancement aims to improve the development experience by making it easier to write and manage asynchronous code in Express.js applications.

## 2. Motivation

The current implementation of Express.js relies heavily on callback functions for handling asynchronous operations. While this approach has served the framework well, it can lead to callback hell and make code harder to read and maintain. With the widespread adoption of async/await and promises in modern JavaScript development, and the introduction of TypeScript, the ecosystem has matured to make promise-based middleware feasible.

## 3. Detailed Design

To implement promise-based middleware in Express.js, we propose the following design:

- **Middleware Function Signature**: Middleware functions can return promises. If a middleware function returns a promise, the Express.js core will wait for the promise to resolve before moving on to the next middleware or the route handler.

- **Error Handling**: Errors can be propagated through the promise chain. If a promise returned by a middleware function or a route handler rejects, Express.js will invoke its error-handling mechanisms.

- **Type Safety**: The implementation will utilize TypeScript to ensure type safety and provide clear interfaces for developers.

- **Backwards Compatibility**: The enhancement will be designed to maintain backwards compatibility with existing synchronous middleware and route handlers.

## 4. Drawbacks

One potential drawback of this enhancement is the added complexity to the Express.js core. However, with modern tooling and language features like TypeScript, this complexity can be managed. Another concern is performance; the implementation will be designed to minimize any performance impact.

## 5. Alternatives

An alternative approach could be to continue using callbacks for asynchronous operations. However, this approach does not leverage the benefits of promises and async/await, which have become standard in modern JavaScript development.
