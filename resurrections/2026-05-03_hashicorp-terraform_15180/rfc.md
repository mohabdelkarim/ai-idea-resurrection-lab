# RFC: Improving perception of changes when showing diff

## Summary
The purpose of this RFC is to propose a new design for the Terraform diff viewer. The current diff viewer is not user-friendly and does not provide enough information about the changes made to the Terraform configuration.

## Motivation
The motivation behind this proposal is to improve the user experience of the Terraform CLI and provide more accurate and relevant information about the changes made to the Terraform configuration.

## Detailed Design
The proposed design involves using a combination of technologies such as React, TypeScript, and the Terraform CDK. The diff viewer would be built as a separate module that can be easily integrated with the Terraform CLI and other tools.

## Drawbacks
One of the drawbacks of this proposal is that it would require significant changes to the Terraform codebase. Additionally, the proposed design would require more resources and maintenance than the current diff viewer.

## Alternatives
One alternative to this proposal is to use an existing diff viewer library or tool. However, this would require significant customization and integration with the Terraform codebase.

## Unresolved Questions
One unresolved question is how to handle default and computed values in the diff viewer. Another question is how to improve the accuracy and relevance of the diff viewer over time.
