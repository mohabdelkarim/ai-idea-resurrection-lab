# RFC: Modal/pop-up widget

## 1. Summary

This RFC proposes the implementation of a modal or pop-up widget in Streamlit, allowing users to display specific data in a focused, overlay window. The modal widget will enhance the user experience by providing a way to present supplementary information without cluttering the main application interface.

## 2. Motivation

Streamlit's current design does not support modal or pop-up windows, limiting the ability to display contextual information or supplementary data. This omission can lead to cluttered interfaces and reduced usability. The proposed modal widget will enable Streamlit users to create more intuitive and user-friendly applications.

## 3. Detailed Design

The modal/pop-up widget will be implemented as a custom Streamlit component, leveraging modern frontend technologies such as React or a Python library like PyQt. The component will consist of the following features:

- A trigger element (e.g., a button) that activates the modal.
- A modal window with a customizable title, content area, and optional close button.
- Seamless integration with Streamlit's existing UI and layout features.

The implementation will involve:

1. Developing a custom Streamlit component using a JavaScript library (e.g., React) or a Python library (e.g., PyQt).
2. Designing a simple and intuitive API for users to create and customize modals.
3. Ensuring cross-platform compatibility and responsiveness.

## 4. Drawbacks

The implementation of a modal/pop-up widget may introduce the following challenges:

- Increased complexity in the Streamlit codebase.
- Potential performance impact due to the addition of a new component.
- Compatibility issues with existing Streamlit features or user-created components.

## 5. Alternatives

Alternative solutions to the modal/pop-up widget include:

- Using existing Streamlit features, such as sidebar panels or expander components, to display supplementary information.
- Implementing a third-party library or component that provides similar functionality.

However, these alternatives may not offer the same level of functionality and user experience as a dedicated modal/pop-up widget.
