# Analysis: Support multiple windows

> Alacritty currently lacks native support for multiple windows, requiring users to open new instances.

**Why it will work now:** Multiple windows in Alacritty would improve user experience for those using window managers instead of tmux.

---

## Why It Died

The issue failed due to lack of resources and prioritization. Alacritty's focus was on improving performance and fixing existing issues. Implementing multiple windows required significant changes to the terminal emulator's architecture.

## Why 2026 Changes Everything

With the advancement of Rust's GUI libraries, such asiced and floem, and the development of cross-platform window management APIs like winit, it's now feasible to implement multiple windows in Alacritty. Additionally, the growing popularity of using terminal emulators with window managers instead of tmux has increased demand for this feature.

## Modern Architecture

The modern design would involve creating a new `WindowManager` class that handles window creation, management, and destruction. This class would utilize the winit API for window management and integrate with Alacritty's existing `Terminal` class. The `Terminal` class would need to be refactored to support multiple instances. The implementation would follow the MVC pattern, with the `WindowManager` acting as the controller.
