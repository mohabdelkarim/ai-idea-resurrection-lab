# RFC: Rule Proposal: prefer-optional-chaining

## 1. Summary

This RFC proposes a new ESLint rule, `prefer-optional-chaining`, which suggests using optional chaining (`?.`) instead of verbose null checks. The rule aims to improve code readability and reduce boilerplate code.

## 2. Motivation

The optional chaining feature, introduced in ECMAScript 2020, provides a concise way to access nested properties without throwing errors when encountering null or undefined values. However, many developers still use verbose null checks, which can lead to cluttered and harder-to-read code.

The ESLint community has shown interest in enhancing its rule set to cover more modern JavaScript features. With the increasing adoption of modern JavaScript features, including optional chaining, and the growing popularity of ESLint, the time is ripe to revisit this proposal.

## 3. Detailed Design

The `prefer-optional-chaining` rule would:

*   Extend the `suggestion` category of rules in ESLint
*   Utilize a parser to identify patterns that can be replaced with optional chaining
*   Provide a clear and actionable recommendation for developers to use the more concise and expressive optional chaining syntax
*   Likely involve creating a new AST (Abstract Syntax Tree) selector and a corresponding fixer to automatically convert the code

Example:

javascript
// Before
if (obj && obj.a && obj.a.b) {
  console.log(obj.a.b);
}

// After
console.log(obj?.a?.b);
## 4. Drawbacks

*   Some developers might not be familiar with the optional chaining feature, which could lead to initial resistance to the rule.
*   The rule might not cover all edge cases, potentially leading to false positives or false negatives.

## 5. Alternatives

*   Instead of creating a new rule, the ESLint team could consider integrating optional chaining suggestions into existing rules, such as `no-unnecessary-conditionals` or `simplify-conditional`.
*   Another approach could be to create a more general rule that suggests using various modern JavaScript features, including optional chaining.
