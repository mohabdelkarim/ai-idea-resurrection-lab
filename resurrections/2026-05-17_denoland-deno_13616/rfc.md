# RFC: [fmt] Option to remove semicolons

## Summary

This RFC proposes adding an option to deno fmt, Deno's code formatting tool, to remove semicolons from JavaScript and TypeScript code. The option, `useSemicolon`, will default to `true` to maintain current behavior and allow users to opt-out of semicolon removal.

## Motivation

The Deno community has grown significantly, and with it, the need for customizable formatting options has become more pressing. Existing tools like Prettier and ESLint offer flexible formatting and linting capabilities, setting a new standard for developer tooling. By introducing the `useSemicolon` option, deno fmt will become more competitive and appealing to developers who prefer semicolon-free code.

## Detailed Design

To implement this feature, we will introduce a new option `useSemicolon` in the deno fmt configuration, which defaults to `true`. When set to `false`, it will utilize dprint's `asi` option to remove automatically insertable semicolons. This design follows the existing pattern of formatting options in deno fmt, such as `useTabs` and `indentWidth`.

The `useSemicolon` option will be exposed through the deno fmt CLI and configuration file. Users can configure it using the following methods:

- CLI flag: `--use-semicolon=false`
- Configuration file: `{ "useSemicolon": false }`

## Drawbacks

One potential drawback of this feature is that it may conflict with existing style conventions in the Deno community. However, by making it an opt-in feature, we can minimize disruptions and allow users to choose their preferred coding style.

## Alternatives

An alternative approach could be to use a separate tool or plugin to handle semicolon removal. However, integrating this feature directly into deno fmt will provide a more seamless experience for users and maintain consistency with existing formatting options.
