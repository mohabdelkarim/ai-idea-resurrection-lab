# RFC: Folder with active storage

Summary

--------

This RFC proposes organizing ActiveStorage files into folders by model. This will improve the management and scalability of file uploads in Rails applications.



Motivation

----------

ActiveStorage provides a convenient way to handle file uploads in Rails applications. However, as the number of uploads grows, it becomes increasingly difficult to manage files in a flat structure. Organizing files into folders by model will improve file management, reduce storage service load, and enhance overall application performance.


The current flat structure can lead to issues such as:

- Difficulty in finding and managing files

- Increased latency due to the load on storage services

- Scalability concerns as the number of files grows


By organizing files into folders, we can:

- Improve file management and organization

- Reduce the load on storage services

- Enhance application performance and scalability



Detailed Design

---------------

To implement this feature, we propose the following:

1. Create a new `Folder` model that will hold the files.

2. Leverage ActiveStorage's `blob` and `service` concepts to create a hierarchical folder structure.

3. Use a combination of the model's id and a custom folder name to create the folder structure.

4. Update the `ActiveStorage::Service` module to handle file uploads to specific folders.

5. Provide a configuration option to enable or disable folder organization.


The folder structure will be as follows:

`#{model_name}/#{model_id}/#{blob_key}`

Where:

- `model_name` is the name of the model (e.g., `User`, `Product`)

- `model_id` is the id of the model instance

- `blob_key` is the key of the blob



Drawbacks

----------

The main drawbacks of this proposal are:

- Additional complexity in the implementation

- Potential issues with backwards compatibility

- Additional storage service requests to create folders


However, these drawbacks can be mitigated by:

- Careful planning and testing of the implementation

- Providing a configuration option to enable or disable folder organization

- Optimizing storage service requests to minimize the impact



Alternatives

------------

Alternative solutions considered include:

- Using a separate storage service for each model

- Using a third-party gem to handle file uploads and organization

However, these alternatives have their own drawbacks, such as:

- Increased complexity and cost

- Vendor lock-in

- Potential issues with integration and compatibility


The proposed solution provides a simple and elegant way to organize files into folders, leveraging the existing ActiveStorage infrastructure.



Unresolved Questions

--------------------

The following questions remain unresolved:

- How to handle files that are already uploaded and not organized into folders?

- How to handle backwards compatibility with existing applications?

- What is the optimal folder structure and naming convention?


These questions will be addressed through further discussion and testing during the implementation phase.
