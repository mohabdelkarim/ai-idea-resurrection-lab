# RFC: Allow to open different windows with different permissions

## Summary

This RFC proposes a feature to allow users to open different windows with different permissions in VSCode. The goal is to provide a secure way for users to run VSCode with elevated privileges, while keeping the main instance of VSCode running with standard privileges.

## Motivation

VSCode is a powerful editor that supports a wide range of extensions, some of which may require elevated privileges to function properly. However, running VSCode with administrator rights can pose a security risk, as it would allow malicious extensions to run with elevated privileges. By allowing users to open different windows with different permissions, we can provide a secure way for users to run VSCode with elevated privileges, while keeping the main instance of VSCode running with standard privileges.

To implement this feature, we propose the following:

* Create a new extension that provides a 'Run as Administrator' command.
* When the 'Run as Administrator' command is executed, use the Electron framework's support for elevated privileges to launch a new instance of VSCode with administrator rights.
* Use VSCode's API to communicate between the two instances and ensure that the administrator instance has the necessary permissions to perform the required actions.
* Implement proper error handling to ensure that the feature is robust and secure.

## Drawbacks

The main drawback of this feature is the potential security risk of running VSCode with elevated privileges. If a malicious extension is installed, it could potentially exploit the elevated privileges to compromise the user's system. To mitigate this risk, we propose to implement strict security checks and validation to ensure that only trusted extensions can run with elevated privileges.

## Alternatives

An alternative approach would be to implement a fine-grained permission system, where extensions can request specific permissions at runtime. However, this approach would require significant changes to the VSCode extension model and may not provide the same level of security as running separate instances with different privileges.
