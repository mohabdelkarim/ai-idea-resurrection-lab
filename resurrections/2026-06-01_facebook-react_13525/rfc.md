# RFC: React Fire: Modernizing React DOM

## Summary

React Fire aims to modernize React DOM by re-aligning it with native DOM behavior and reducing technical debt. This effort was previously abandoned due to complexity and potential breaking changes, but advancements in modern tooling and the ecosystem make it more feasible today.

## Motivation

The React DOM event system and architecture have accumulated technical debt over the years, leading to complexity and bundle size issues. The original implementation of React DOM was designed to work around browser inconsistencies and provide a simplified programming model. However, the web platform has evolved, and modern browsers have converged on standard behaviors. By modernizing React DOM, we can reduce the maintenance burden, improve performance, and make it easier for developers to understand and use React.

## Detailed Design

The modern design for React Fire involves re-examining the React DOM architecture to better align it with the DOM's native behavior. Key changes include:

* Revisiting the event system to reduce complexity and bundle size
* Stopping the reflection of input values in the `value` attribute, which was originally added to match the `value` JSX attribute but caused a cascade of bugs
* A more streamlined approach to handling DOM updates, leveraging React's concurrent rendering capabilities
* Improved support for React Server Components
* Better code maintainability and scalability through the use of TypeScript

## Drawbacks

The main drawbacks of this proposal are the potential for breaking changes and the effort required for migration. However, with the help of automated codemod tools and the growth of the React community, we believe that these challenges can be mitigated.

## Alternatives

One alternative to React Fire is to continue with the current architecture and incrementally address specific pain points. However, this approach would not address the underlying technical debt and would likely lead to continued complexity and maintenance issues. Another alternative is to explore a complete rewrite of React DOM, but this would be a much more significant undertaking and may not be feasible in the short term.
