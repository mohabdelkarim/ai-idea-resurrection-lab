# RFC: Ignore changes for a single AWS Resource Tag

## Summary
This RFC proposes the addition of a new configuration option, ignore_tag_changes, to the Terraform AWS provider. This option would allow users to specify a list of tags to ignore changes for, providing more fine-grained control over resource tag management.

## Motivation
The current implementation of the ignore_changes argument in Terraform only allows users to ignore all changes to a resource or none at all. This can be limiting in scenarios where users want to ignore changes to specific tags, such as the VERSION tag, which is updated regularly by Jenkins. The proposed feature would provide a more flexible and user-friendly way to manage resource tags.

## Detailed Design
The proposed feature would introduce a new configuration option, ignore_tag_changes, which would be a list of strings representing the tags to ignore changes for. The Terraform provider would then use this option to filter the resource tags and only update the tags that are not ignored. The design would involve creating a new Terraform resource, aws_resource_tag_ignore, which would handle the ignore_tag_changes configuration and interact with the AWS API to filter the resource tags.

## Drawbacks
One potential drawback of this feature is that it may add complexity to the Terraform configuration language. Additionally, the feature may require significant changes to the Terraform provider architecture, which could be time-consuming and error-prone.

## Alternatives
One alternative to this feature is to use the existing ignore_changes argument and specify all the tags that should not be ignored. However, this approach can be cumbersome and prone to errors, especially when dealing with a large number of tags.

One unresolved question is how to handle cases where the ignore_tag_changes option is not specified. Should the Terraform provider default to ignoring all changes or none at all?
