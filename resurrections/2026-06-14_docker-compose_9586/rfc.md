# RFC: "Custom build outputs" support

# Summary

This RFC proposes adding support for custom build outputs in Docker Compose. The goal is to introduce a new configuration option that allows users to specify custom outputs for their builds, enabling more flexibility and extensibility in the build process.




# Motivation

Docker Compose currently supports building and running multi-container Docker applications. However, the build process is limited to producing a single Docker image. With the advancements in Docker and containerization technologies, there is a growing need for more flexible build outputs. Custom build outputs can enable use cases such as:

* Exporting images to multiple registries
* Generating additional artifacts, such as SBOMs or vulnerability reports
* Pushing images to different storage systems

The introduction of Docker BuildKit and its API provides a foundation for implementing custom build outputs. By leveraging BuildKit's API, Docker Compose can extend its build process to support custom outputs.




# Detailed Design

To implement custom build outputs in Docker Compose, we will:

1. Introduce a new configuration option in the Compose file, `build.outputs`, which will allow users to specify custom build outputs.
2. Create a new module in Docker Compose to handle BuildKit API interactions and integrate it with the existing build process.
3. Update the build process to use BuildKit's API for building and exporting images.

The `build.outputs` configuration option will be a list of output configurations, where each configuration specifies the output type, destination, and other relevant settings. For example:

services:
  myservice:
    build:
      context: .
      outputs:
        - type: docker
          destination: myregistry/myimage:latest
        - type: sbom
          destination: /path/to/sbom.json



# Drawbacks

The main drawbacks of this proposal are:

* Additional complexity in the build process
* Potential performance impact due to the introduction of new API calls
* Need for additional testing and validation to ensure compatibility with different use cases

However, the benefits of custom build outputs, such as increased flexibility and extensibility, outweigh the drawbacks.




# Alternatives

Another alternative is to rely on external tools, such as shell scripts or Makefiles, to achieve custom build outputs. However, this approach would require users to manage additional complexity and may lead to inconsistencies across different environments.
