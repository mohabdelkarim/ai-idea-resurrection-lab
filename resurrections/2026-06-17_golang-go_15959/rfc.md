# RFC: cmd/go: go get download progress and download speed status 

# Summary

This RFC proposes to enhance the `go get` command to display download progress and speed. The feature will provide users with a better understanding of the download process, especially for large packages or slow internet connections.




# Motivation

The `go get` command is a crucial tool for Go developers to download and manage packages. However, it currently lacks a feature to display download progress and speed. This information is essential for users to estimate the remaining time for the download to complete, especially when dealing with large packages or slow internet connections. The proposed feature aims to improve the user experience by providing a clear and concise progress indicator.




# Detailed Design

The proposed feature will be implemented by modifying the `go get` command to track and display download progress and speed. The implementation will involve:

*   Utilizing the `net/http` package to track the progress of downloads.
*   Displaying the progress using a library like `github.com/charmbracelet/bubbletea` for terminal UI.
*   Calculating the percentage of the downloaded content and the speed of download in bytes per second.
*   Designing the feature to be configurable, allowing users to opt-out of the progress display if needed.

The progress display will include the following information:

*   The package name being downloaded.
*   The total size of the package.
*   The downloaded size so far.
*   The percentage of the package downloaded.
*   The download speed in bytes per second.
*   An estimated time of completion.




# Drawbacks

The proposed feature may have the following drawbacks:

*   Additional dependencies: The implementation will require an additional dependency on a library like `github.com/charmbracelet/bubbletea` for terminal UI.
*   Performance overhead: Tracking download progress and speed may introduce a minor performance overhead due to the additional calculations and display updates.

However, these drawbacks are expected to be minimal and can be mitigated by:

*   Using a lightweight library for terminal UI.
*   Optimizing the implementation to minimize performance overhead.




# Alternatives

Alternative solutions to display download progress and speed include:

*   Using an existing library or tool to display progress bars, such as `github.com/schollz/progressbar/v3`.
*   Implementing a custom progress bar using only the `net/http` package and standard Go libraries.

However, using a dedicated library like `github.com/charmbracelet/bubbletea` is expected to provide a more user-friendly and customizable progress display.
