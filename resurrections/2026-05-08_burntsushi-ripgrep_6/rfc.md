# RFC: add support for mercurial

# Summary

This RFC proposes adding support for the Mercurial version control system to the ripgrep project. The goal is to enable ripgrep to ignore files and directories specified in Mercurial's `.hgignore` files, allowing users to efficiently search for patterns in Mercurial repositories.




# Motivation

The motivation behind this RFC is to enhance the usability of ripgrep for developers who use Mercurial as their version control system. Currently, ripgrep only supports Git repositories, leaving Mercurial users without a robust searching tool that respects their version control ignore rules. By adding Mercurial support, we can expand ripgrep's user base and provide a more comprehensive searching experience.




# Detailed Design

The implementation will involve the following steps:

*   Create a new `MercurialIgnore` struct that handles the parsing of `.hgignore` files and subincludes.
*   Utilize the `ignore` crate to translate the patterns in `.hgignore` files into regexes that can be used by ripgrep.
*   Use the `mercurial-rs` crate to interact with Mercurial repositories and retrieve the ignore patterns.
*   Define a trait for different version control systems, allowing for a unified interface for handling ignore rules.

The `MercurialIgnore` struct will be responsible for:

*   Reading and parsing `.hgignore` files
*   Handling subincludes and translating patterns into regexes
*   Providing an interface for retrieving ignore rules

The `mercurial-rs` crate will be used to:

*   Open and interact with Mercurial repositories
*   Retrieve the root directory of the repository
*   Access the `.hgignore` files and subincludes

The `ignore` crate will be used to:

*   Translate the patterns in `.hgignore` files into regexes
*   Handle the complexity of Mercurial's ignore patterns




# Drawbacks

The main drawbacks of this implementation are:

*   The added complexity of handling Mercurial's `.hgignore` files, which support both regexes and globs, as well as subincludes.
*   The potential performance impact of adding an additional version control system to ripgrep.
*   The requirement to maintain and update the Mercurial support as the `mercurial-rs` crate and Mercurial itself evolve.




# Alternatives

The alternatives to this implementation are:

*   Using an existing library or tool that provides Mercurial support, such as `hg-rust`, but this may not provide the required level of integration with ripgrep.
*   Implementing a custom solution for handling `.hgignore` files, but this may be more error-prone and less maintainable.
*   Leaving Mercurial support out of ripgrep, but this would limit the usability of ripgrep for Mercurial users.




# Unresolved Questions

The following questions remain unresolved:

*   How will we handle cases where the `.hgignore` file is not in the root directory of the repository?
*   How will we handle cases where the repository has multiple `.hgignore` files with conflicting rules?
*   What is the performance impact of adding Mercurial support to ripgrep, and are there any potential optimizations that can be made?
