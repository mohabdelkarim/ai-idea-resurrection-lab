# RFC: Name of task containing block is not printed to stdout

### Summary

This RFC proposes a solution to ensure that Ansible blocks print the name of the task containing the block to stdout. Currently, when a block is executed, the name of the task that contains the block is not displayed, making it difficult to track the progress and identify the task being executed.

### Motivation

The motivation behind this proposal is to improve the visibility and transparency of task execution in Ansible. By displaying the name of the task containing the block, users can better understand the execution flow and debug their playbooks more effectively. This feature is particularly useful for complex playbooks with nested blocks, where it can be challenging to identify the current task being executed.

### Detailed Design

To implement this feature, we will modify the block task executor to append the task name to the block's child tasks. This can be achieved by introducing a new plugin or modifying the existing block executor to dynamically generate task names. The solution will involve updating the task executor and possibly adding new configuration options to control the behavior.

The proposed changes include:

*   Modifying the `TaskExecutor` class to include the task name in the block's child tasks
*   Introducing a new plugin, `task_name_inheritor`, which will handle the task name inheritance for blocks
*   Updating the `playbook_executor` to use the new plugin and display the task name

The changes will be made in the following files:

*   `lib/ansible/executor/task_executor.py`
*   `lib/ansible/plugins/strategy/__init__.py`
*   `lib/ansible/playbook/playbook_executor.py`

### Drawbacks

The potential drawbacks of this proposal include:

*   Performance impact: The additional processing required to generate task names may introduce a performance overhead
*   Complexity: The changes may add complexity to the Ansible codebase, making it harder to maintain and debug

The alternative solutions considered include:
