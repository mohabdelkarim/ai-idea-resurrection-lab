# RFC: proposal: sync, sync/atomic: add PoolOf, MapOf, ValueOf

## 1. Summary

This RFC proposes the addition of type-safe PoolOf, MapOf, and ValueOf to the sync and atomic packages in the Go standard library, leveraging Go generics for compile-time type safety.

## 2. Motivation

The current sync.Pool, sync.Map, and atomic.Value types in Go provide low-level, type-unsafe mechanisms for managing temporary objects, concurrent maps, and atomic values, respectively. While they are powerful, their lack of type safety can lead to runtime errors and makes the code harder to maintain.

With the introduction of Go generics in Go 1.18, it is now technically feasible to create type-safe variants of these types. This proposal aims to enhance the standard library by providing PoolOf, MapOf, and ValueOf types that ensure type safety at compile time.

## 3. Detailed Design

The proposal involves adding the following types to the sync and atomic packages:

- `sync.PoolOf[T any]`: A type-safe pool for temporary objects of type T. It would provide methods similar to sync.Pool but with generic type safety.
- `sync.MapOf[K comparable, V any]`: A type-safe map for concurrent access, where K is the key type and V is the value type. It would offer methods akin to sync.Map but with the added benefit of compile-time type checking.
- `atomic.ValueOf[T any]`: A type-safe wrapper for atomic.Value, allowing for atomic load and store operations on values of type T.
