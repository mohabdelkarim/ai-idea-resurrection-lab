# RFC: Remove restrictions of `gh pr view --json files`

Summary

--------

This RFC proposes removing the 100 record limit for 'gh pr view --json files' by implementing GraphQL pagination. The goal is to enable fetching all files associated with a pull request, regardless of the number of files.


Motivation

----------

The current implementation of 'gh pr view --json files' has a hardcoded limit of 100 records. This limitation causes issues when dealing with large pull requests that have over 500 files. The introduction of GitHub's GraphQL API pagination and the Go library 'github.com/shurcooL/github_flate' provides a feasible solution to fetch large datasets in chunks, making it possible to remove the 100 record limit.


Detailed Design

---------------

The solution involves using the GitHub GraphQL API with pagination to fetch all files associated with a PR. This can be achieved by implementing a new function, 'fetchFiles', which takes a GitHub client, PR number, and file accumulator as arguments. The function will use the 'github_flate' library to handle pagination and accumulate files.



The 'fetchFiles' function will:

1. Initialize a file accumulator to store the fetched files.

2. Use the GitHub GraphQL API to fetch the first batch of files (e.g., 100 records).

3. Check if the response contains a 'pageInfo' with a 'hasNextPage' flag set to true.

4. If 'hasNextPage' is true, use the 'github_flate' library to fetch the next batch of files.

5. Repeat steps 3-4 until all files have been fetched.

6. Return the accumulated files.


Drawbacks

----------

The main drawback of this solution is the potential increase in API requests and latency when dealing with very large pull requests. However, this can be mitigated by implementing efficient pagination and batching logic.



Another potential drawback is the added complexity of the 'fetchFiles' function, which may require additional testing and maintenance.


Alternatives

------------

An alternative solution could be to increase the hardcoded limit of 100 records to a higher number. However, this would still be a temporary solution and might not accommodate extremely large pull requests.



Another alternative is to use the GitHub REST API instead of the GraphQL API. However, the GraphQL API provides more flexibility and efficiency in fetching specific data, making it a more suitable choice for this use case.
