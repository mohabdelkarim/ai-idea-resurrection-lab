# RFC: Search based on file timestamps (atime, ctime, mtime)

## Summary

This RFC proposes the implementation of timestamp-based file search in the `fd` command-line utility. The goal is to add support for searching files based on their access time (`atime`), creation time (`ctime`), and modification time (`mtime`) using command-line flags similar to `find`'s `-atime`, `-ctime`, and `-mtime` flags.

## Motivation

The `fd` utility is a popular alternative to `find` for searching files. However, it currently lacks support for timestamp-based search, which is a crucial feature for many users. By implementing this feature, `fd` will become more versatile and competitive with `find`.

## Detailed Design

The implementation will involve the following steps:

*   Add command-line flags for `-atime`, `-ctime`, and `-mtime` to the `fd` command.
*   Parse file timestamps using the `std::fs` module and `tokio` for async I/O operations.
*   Create a `struct` to represent file metadata, including timestamps.
*   Implement a `trait` for filtering files based on timestamps.
*   Integrate the filtering logic with the existing `fd` search functionality.

The command-line flags will accept a value in the format of `+n`, `-n`, or `n`, where `n` is a number representing days. For example:

*   `-atime +7` will find files accessed more than 7 days ago.
*   `-atime -7` will find files accessed less than 7 days ago.
*   `-atime 7` will find files accessed exactly 7 days ago.

## Drawbacks

The implementation of timestamp-based search may introduce some drawbacks:

*   Performance impact: Parsing file timestamps can be slow, especially for large directories. However, this can be mitigated by using async I/O operations and optimizing the filtering logic.
*   Platform inconsistencies: The availability and accuracy of file timestamps can vary across platforms. The implementation will need to account for these differences.
