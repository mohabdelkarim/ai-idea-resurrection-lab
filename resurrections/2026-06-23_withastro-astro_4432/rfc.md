# RFC: Supporting CSS-in-JS

## Summary

Astro should provide native support for popular CSS-in-JS libraries to enhance developer experience and flexibility. This RFC proposes a modular approach to integrate CSS-in-JS libraries into Astro's build process and server-side rendering (SSR) handling.

## Motivation

The Astro community has shown growing interest in using CSS-in-JS libraries. However, current support is limited due to complexities in handling SSR and production build errors with libraries like styled-components, emotion, and linaria. Recent advancements in Astro's architecture, particularly with Astro 4 and Vite's improved SSR capabilities, make it feasible to address these challenges.

## Detailed Design

To implement CSS-in-JS support, the following steps are proposed:

1. **Plugin System**: Develop a plugin system for CSS-in-JS libraries. This will allow library maintainers to create plugins that integrate their libraries with Astro.
2. **Vite External Heuristics**: Enhance Vite's external heuristics to better handle CSS-in-JS libraries.
3. **SSR Handling**: Adapt Astro's SSR handling to support CSS-in-JS libraries. This includes ensuring seamless client-side and server-side rendering.
4. **Modular Design**: Implement a modular design with clear interfaces for library integration. This will enable easy addition of new libraries and reduce maintenance overhead.
5. **Documentation and Testing**: Provide comprehensive documentation for library maintainers and Astro users. Establish a community-driven testing process to ensure compatibility with various libraries.

## Drawbacks

1. **Increased Complexity**: Adding CSS-in-JS support will increase the complexity of Astro's build process and SSR handling.
2. **Maintenance Overhead**: Supporting multiple CSS-in-JS libraries will require additional maintenance efforts.

## Alternatives

1. **Community-Driven Solutions**: Instead of native support, Astro could encourage community-driven solutions, such as third-party plugins or libraries.
2. **Limited Support**: Astro could provide limited support for a select few CSS-in-JS libraries, prioritizing the most popular ones.

## Unresolved Questions

2. **Plugin Development**: What guidelines and tooling will be provided for library maintainers to develop plugins?
