# RFC: Add utility to synchronously inspect promise state

## Summary

This RFC proposes the addition of a utility method to synchronously inspect the state of a Promise in Node.js. The method, `util.promiseState(promise)`, would return a string indicating whether the promise is 'pending', 'fulfilled', or 'rejected'. This addition aims to provide developers with a stable, native API to inspect promise states, which is beneficial for performance optimizations, testing, and debugging.

## Motivation

The existing internal method `process.binding('util').getPromiseDetails(promise)` was removed in Node.js 16, leaving developers without a straightforward, stable way to inspect promise states. The community-created package `p-state` attempts to parse the output of `util.inspect()`, but this approach is fragile and not ideal. With the evolution of async/await and promise usage in modern Node.js development, having a synchronous utility to inspect promise states is increasingly important.

## Detailed Design

The proposed `util.promiseState(promise)` method would be part of the `util` module, making it easily accessible to developers. The method would take a Promise as an argument and return a string indicating the promise state:
- 'pending': The promise has not yet settled.
- 'fulfilled': The promise has successfully resolved.
- 'rejected': The promise has rejected with an error.

The implementation could leverage the `util.inspect()` function or other internal Node.js mechanisms to determine the promise state. It would handle various promise types, including native promises and those from libraries.

## Drawbacks

1. **Additional Maintenance**: Adding a new method to the `util` module increases the maintenance surface. The method's implementation must be kept in line with evolving promise specifications and internal Node.js mechanisms.

2. **Potential for Misuse**: Providing a synchronous method to inspect promise states could lead to misuse in performance-critical code paths, where asynchronous programming is preferred.
