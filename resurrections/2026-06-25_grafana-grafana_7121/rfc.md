# RFC: [Feature request] Email templating in Alert engine

# Summary

This RFC proposes the implementation of an email templating engine for Grafana's alerting feature. The engine will enable users to create customizable email alerts using a templating language, providing more flexibility and control over alert notifications.




# Motivation

Grafana's alerting feature is widely used to notify users of critical events and anomalies in their data. However, the current implementation of email alerts lacks customization options, making it difficult for users to tailor notifications to their specific needs. The proposed email templating engine will address this limitation, allowing users to create personalized email alerts that include relevant data and context.




# Detailed Design

The email templating engine will be built using Go templates, leveraging the Go `text/template` package. The engine will consist of the following components:

*   Template Registry: A centralized registry to store and manage email templates.
*   Data Model: A data model to provide context for templating, including metrics, dashboard variables, and alert metadata.
*   Templating Engine: A templating engine that uses Go templates to render email templates.

The templating engine will support various data types, including metrics, dashboard variables, and alert metadata. Users will be able to create and manage templates using a user-friendly interface, and integrate them with existing Grafana features, such as data source plugins and alerting capabilities.




# Drawbacks

The implementation of the email templating engine may introduce additional complexity to the Grafana alerting feature, potentially affecting performance and maintainability. Furthermore, the use of Go templates may require additional learning and adaptation for users who are not familiar with the templating language.




# Alternatives

Alternative solutions to the email templating engine include:

*   Using an existing templating library or engine, such as Jinja2 or Mustache.
*   Implementing a custom templating solution using a different programming language or technology.
*   Providing a limited set of pre-defined templates and placeholders for users to customize.

However, using Go templates provides a robust and maintainable solution that aligns with Grafana's existing technology stack.
