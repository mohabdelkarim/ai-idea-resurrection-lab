# RFC: Allow specifying JSON stringifier

Summary

--------

This RFC proposes adding an optional 'stringifier' option to the res.json() method in Express.js, allowing users to specify a custom JSON stringifier function.


Motivation

-----------

The res.json() method in Express.js uses JSON.stringify() to stringify response data. However, this can lead to issues when dealing with large integers, as JSON.stringify() will truncate or lose precision. By allowing users to specify a custom JSON stringifier, such as json-bigint.stringify, we can improve the handling of large integers and provide more flexibility for users with specific use cases.


Detailed Design

----------------

We propose adding an optional 'stringifier' option to the res.json() method. This option would allow users to pass a custom stringifier function, such as json-bigint.stringify. If no stringifier is provided, the method will fall back to using JSON.stringify(). The implementation would involve updating the res.json() method to accept the new option and using it to stringify the response data.



The API would be updated as follows:

res.json(data, options)

Where options is an object with the following properties:

- stringifier: a custom JSON stringifier function (default: JSON.stringify)



Example usage:

const express = require('express');

const app = express();

const jsonBigint = require('json-bigint');



app.get('/example', (req, res) => {

  const data = { largeInteger: 12345678901234567890 };

  res.json(data, { stringifier: jsonBigint.stringify });

});


Drawbacks

----------

The main drawback of this proposal is the potential performance impact of allowing custom stringifiers. However, this can be mitigated by providing a default stringifier that falls back to JSON.stringify(). Additionally, users who require custom stringification can opt-in to using the feature.



Another potential drawback is the added complexity of the res.json() method. However, this can be managed by keeping the API simple and providing clear documentation.


Alternatives

------------

One alternative to this proposal is to add a new method, such as res.jsonBigint(), that uses a custom stringifier. However, this would lead to a proliferation of methods and make the API more complex. Another alternative is to use a middleware to handle custom stringification, but this would add extra overhead and make the code more difficult to read.
