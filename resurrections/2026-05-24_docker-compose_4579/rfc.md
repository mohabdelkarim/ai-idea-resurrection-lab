# RFC: Using scale number to adjust mounted volumes?

## Summary

This RFC proposes the addition of a feature to Docker Compose that allows dynamic volume mounting using scale numbers. The goal is to enable users to adjust the number of mounted volumes based on the scale of a service.

## Motivation

Currently, Docker Compose requires a fixed number of volume mounts to be defined in the `docker-compose.yml` file. However, in many scenarios, the number of volumes needed depends on the scale of the service. For example, in a load-balanced web server setup, the number of mounted volumes may need to be adjusted based on the number of replicas.

## Detailed Design

The proposed feature will utilize the existing templating and variable interpolation mechanisms in Docker Compose. A new class, `VolumeInterpolator`, will be introduced to handle the interpolation of volume mounts. The `docker-compose` API will be extended to support templating in volume definitions.

The feature will be implemented as follows:

*   Introduce a new syntax for volume definitions that allows templating, e.g., `${VOLUME_SCALE}`.
*   Create a `VolumeInterpolator` class that will handle the interpolation of volume mounts based on the scale number.
*   Extend the `Service` class to support dynamic volume mounting.
*   Update the `docker-compose` API to support templating in volume definitions.

## Drawbacks

The proposed feature may introduce additional complexity to the Docker Compose configuration file. There are also potential security concerns if not implemented properly, such as allowing arbitrary volume mounts.

## Alternatives

Alternative solutions include:

*   Using environment variables to adjust the number of volume mounts.
*   Utilizing existing templating mechanisms to define dynamic volume mounts.

However, these alternatives may not provide the same level of flexibility and ease of use as the proposed feature.
