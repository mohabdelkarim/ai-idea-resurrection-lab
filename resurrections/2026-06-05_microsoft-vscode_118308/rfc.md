# RFC: Move away from electron

## Summary

This RFC proposes migrating the Visual Studio Code (VSCode) desktop application from the Electron framework to Tauri. This migration aims to improve performance, security, and usability, particularly on low-end devices. The proposal involves leveraging Tauri's Rust-based core for the application shell while maintaining compatibility with the existing TypeScript-based extension framework.

## Motivation

VSCode currently uses Electron, which, despite its popularity, has limitations in terms of performance and security. Electron's architecture, based on Chromium and Node.js, can result in high memory usage and potential security vulnerabilities. In contrast, Tauri, built on Rust and WebAssembly (WASM), offers a more lightweight and secure alternative. Tauri's recent stability improvements and the growing importance of performance and security in desktop applications make this migration timely. By moving to Tauri, VSCode can benefit from improved resource efficiency, enhanced security posture, and a better user experience on lower-end hardware.

1. **Tauri Core Integration**: Replace the Electron shell with Tauri's Rust-based core. This core provides a high-performance, secure foundation for the VSCode application.

2. **Extension API Wrapper**: Develop a Tauri-compatible wrapper for the VSCode extension API. This wrapper must ensure seamless integration with existing extensions, allowing them to function without modification.

3. **TypeScript Extension Support**: Explore using WebAssembly (WASM) to run TypeScript-based extensions within the Tauri context. This approach would provide a secure and performant environment for extensions, leveraging the strengths of both Tauri and the existing extension framework.

4. **Testing and Validation**: Implement comprehensive testing to validate the functionality and performance of VSCode on Tauri. This includes compatibility testing for extensions, performance benchmarks, and security audits.
