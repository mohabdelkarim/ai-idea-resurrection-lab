# RFC: Feature Request: terraform fmt applies sort to variables.tf and terraform.tfvars

# Summary

This RFC proposes the addition of a sorting feature to the `terraform fmt` command, specifically targeting `variables.tf` and `terraform.tfvars` files. The goal is to provide a consistent and automated way to sort the contents of these files, improving the overall readability and maintainability of Terraform configurations.




# Motivation

As Terraform configurations grow in complexity, the need for organized and readable code becomes increasingly important. Currently, `terraform fmt` focuses on formatting HCL files according to the Terraform style guide. However, the sorting of variable definitions within `variables.tf` and `terraform.tfvars` files is not addressed. This leads to inconsistencies in code organization, making it harder for users to manage and understand their configurations.

The primary motivations for this feature are:
1. **Improved Readability**: Sorted variable definitions make configurations easier to read and understand.
2. **Consistency**: Enforces a standard ordering, reducing inconsistencies across different parts of a configuration.
3. **Efficiency**: Automates the sorting process, saving users time and effort.




# Detailed Design

## Overview

The proposed feature will be implemented as a subcommand within `terraform fmt`, named `sort`. This subcommand will recursively traverse a given directory, identify `variables.tf` and `terraform.tfvars` files, and sort their contents alphabetically.

## Implementation

1. **Parsing and Manipulation**: Utilize the existing HCL parsing library in Go to read and manipulate the variable definition files.
2. **Sorting Logic**: Implement a sorting algorithm that can handle the specific HCL structure of variable definitions. The sorting should be case-sensitive and alphabetically ordered by variable name.
3. **Subcommand Integration**: Integrate the `sort` subcommand into `terraform fmt`, allowing users to invoke the sorting functionality easily.

## Example Usage

sh
terraform fmt sort
This command will sort the variable definitions in `variables.tf` and `terraform.tfvars` files within the current working directory and its subdirectories.




# Drawbacks

1. **Backward Compatibility**: The sorting feature might change the formatting of existing configurations, potentially causing issues if not properly tested.
2. **Performance**: For very large configurations, the sorting process could take noticeable time, impacting user experience.
3. **Configuration Management**: Users might need to adapt their configuration management processes to accommodate the automated sorting feature.




# Alternatives

1. **External Tools**: Users could rely on external tools or scripts to sort their variable definitions. However, integrating this functionality directly into `terraform fmt` provides a more streamlined and consistent experience.
2. **Manual Sorting**: Users could manually sort their variable definitions. However, this approach is prone to errors and inconsistencies, especially in large configurations.




# Unresolved Questions

1. **Sorting Algorithm**: The exact sorting algorithm (e.g., case-sensitive, case-insensitive) has not been finalized and should be determined based on user feedback and requirements.
