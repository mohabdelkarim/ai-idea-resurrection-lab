# RFC: Refreshing the VS Code product icon

This RFC proposes refreshing the VS Code product icon to improve visibility and consistency across platforms. The goal is to adopt a modern, Fluent Design-aligned icon that enhances the overall user experience and makes VS Code more recognizable.

With the advancements in design tools and APIs, such as Microsoft's updated Fluent Design System, refreshing the icon is now more feasible and appealing. A modern icon will help improve visibility, consistency, and overall user experience.

## 3. Detailed Design

The proposed icon design will be based on the Fluent Design System, which emphasizes simplicity, clarity, and consistency. The new icon will likely feature:

* A simplified, high-contrast design with added depth for better legibility
* A more minimalist approach to align with recent Microsoft product icons
* A design that is optimized for various platforms, including Windows, macOS, and Linux

The icon will be designed using modern tools like Sketch or Figma and implemented using TypeScript and VS Code's existing icon rendering mechanisms.

## 4. Drawbacks

Potential drawbacks of refreshing the VS Code product icon include:

* Familiarity: Users may be accustomed to the current icon and might find the change disorienting
* Brand recognition: A new icon could affect the recognition of the VS Code brand
* Technical challenges: Implementing the new icon design might require updates to the VS Code codebase and could introduce technical debt

Alternatives to refreshing the VS Code product icon include:
