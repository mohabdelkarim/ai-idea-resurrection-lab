# RFC: {#range ...} block

# Summary

This RFC proposes the introduction of a new block syntax, {#range ...}, to simplify iteration in Svelte. The proposed syntax aims to provide a more concise and readable way to iterate over ranges, similar to Ruby's range operator.




# Motivation

Currently, iterating over ranges in Svelte requires using {#each} loops with array methods like Array.from() or a custom loop. This can lead to verbose code and make it harder to read and maintain. A range block feature would allow developers to write more concise and expressive code.




# Detailed Design

The proposed syntax for the range block is:
{#range variable=start to end}
  <!-- content -->
{/range}
The Svelte compiler would translate this syntax into an equivalent {#each} loop. For example:
{#range i=1 to 5}
  <p>{i}</p>
{/range}
Would be compiled to:
{#each Array.from({length: 5}, (_, i) => i + 1) as i}
  <p>{i}</p>
{/each}
The range block would support various use cases, including:

* Iterating over numbers: {#range i=1 to 10}
* Iterating over letters: {#range letter=a to z}
* Decrementing ranges: {#range i=10 to 1}

The implementation would involve modifying the Svelte parser and compiler to recognize and process the new syntax.




# Drawbacks

One potential drawback of this feature is the added complexity to the Svelte compiler and parser. Additionally, the range block syntax may not be immediately familiar to all developers, which could lead to a learning curve.




# Alternatives

An alternative approach could be to provide a built-in range function that can be used with {#each} loops. However, this would not provide the same level of conciseness and readability as a dedicated range block syntax.




# Unresolved Questions

* How to handle edge cases, such as iterating over non-numeric ranges (e.g., letters)?
* How to support decrementing ranges?
* What is the best way to handle range validation (e.g., ensuring start is less than or equal to end)?
