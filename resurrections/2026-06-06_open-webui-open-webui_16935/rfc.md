# RFC: feat: Support new Image Generation base64 with OpenRouter

# Summary

This RFC proposes to add support for OpenRouter's new image generation feature with base64 output in Open-WebUI. The goal is to enable users to generate images using the google/gemini-2.5-flash-image-preview model and display them in the chat interface.




# Motivation

The motivation behind this proposal is to enhance the user experience in Open-WebUI by integrating image generation capabilities. With the growing adoption of AI models like google/gemini-2.5-flash-image-preview, it is now more feasible to implement this feature. Modern tooling and libraries provide better support for base64 encoding and decoding, making it easier to handle image generation outputs.




# Detailed Design

To implement this feature, we propose the following:

*   Create a new API endpoint under the settings -> Images section that allows users to input their OpenRouter API credentials.
*   The frontend can then send requests to this endpoint to generate images using the google/gemini-2.5-flash-image-preview model.
*   The output will be displayed in the chat interface as a base64-encoded image.
*   The implementation will involve creating a new backend API to handle image generation requests and integrate with the OpenRouter API.
*   We will use modern libraries and tooling to handle base64 encoding and decoding.




# Drawbacks

The potential drawbacks of this proposal are:

*   Additional complexity in the backend API to handle image generation requests and integrate with the OpenRouter API.
*   Potential security risks associated with handling user credentials and sensitive data.
*   The need for additional testing and validation to ensure the feature works as expected.




# Alternatives

The alternatives to this proposal are:
