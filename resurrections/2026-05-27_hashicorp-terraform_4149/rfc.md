# RFC: Partial/Progressive Configuration Changes

## Summary

This RFC proposes the implementation of partial/progressive configuration changes in Terraform. ## Motivation

Currently, Terraform requires a complete and accurate configuration to apply changes. However, in real-world scenarios, configurations can be complex and may need to be applied in a progressive manner. The existing `-target` argument provides a workaround, but it has limitations and can lead to inconsistencies. With the advancements in Terraform's provider protocol and recent improvements in the planning and apply phases, the groundwork has been laid for implementing partial configuration changes.
