# RFC: Better <think> block rendering for DeepSeek-R1 and similar

### 1. Summary

This RFC proposes an improved rendering of `<think>` blocks in LLM (Large Language Model) output within the Open WebUI project. The goal is to make these blocks collapsible and visually distinct from regular text responses, enhancing the overall user experience.

### 2. Motivation

The current rendering of `<think>` blocks in LLM output is not visually distinct and can be overwhelming for users. These blocks often contain intermediate thoughts or reasoning steps that are not essential for understanding the final response. By making them collapsible and visually distinct, we can improve the readability and usability of the LLM output.

### 3. Detailed Design

The proposed design involves creating a custom React component for rendering `<think>` blocks. This component will be a collapsible panel or accordion that contains the `<think>` block content. The component will utilize a modern UI library such as Material-UI or Tailwind CSS to ensure a consistent and visually appealing design.

Key features of the design:

*   Collapsible functionality: Users can toggle the visibility of the `<think>` block, with an expand/collapse indicator.
*   Visually distinct design: The `<think>` block will have a distinct background color, border, or padding to differentiate it from regular text responses.

Implementation details:

*   The component will be built using React and will be compatible with the existing Open WebUI architecture.
*   The component will accept the `<think>` block content as a prop and will render it within the collapsible panel or accordion.

### 4. Drawbacks

Potential drawbacks of this design include:

*   Additional complexity: The introduction of a new custom component may add complexity to the codebase.
*   Performance impact: The use of a collapsible panel or accordion may have a minor performance impact, particularly if the component is not optimized.

### 5. Alternatives

Alternative solutions considered include:

*   Simple text formatting: Applying simple text formatting, such as italics or a different font color, to distinguish `<think>` blocks from regular text responses. However, this approach may not provide sufficient visual distinction and does not offer collapsible functionality.
*   A separate section: Rendering `<think>` blocks in a separate section or sidebar. However, this approach may require significant changes to the existing layout and may not be feasible.

### 6. Unresolved Questions

The following questions remain unresolved:

*   What is the optimal design for the collapsible panel or accordion? Should it be a simple expand/collapse button or a more complex design with animations?
*   How will the component handle edge cases, such as very long or very short `<think>` blocks?
*   What are the performance implications of implementing this feature, and how can we mitigate any potential impact?
