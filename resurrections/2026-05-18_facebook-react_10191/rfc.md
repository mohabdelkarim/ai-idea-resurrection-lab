# RFC: Consider re-licensing to AL v2.0, as RocksDB has just done

## 1. Summary

This RFC proposes re-licensing the React.js library under the Apache License v2.0 (AL v2.0), aligning with the licensing standards of the Apache Software Foundation (ASF) and following the recent example set by RocksDB. This change aims to alleviate compatibility issues with ASF projects and improve the overall licensing clarity for React users.

## 2. Motivation

The React.js library is widely used across various projects, including those within the Apache Software Foundation. However, React's current licensing may pose compatibility challenges with ASF projects, which typically require AL v2.0 or similar permissive licenses. 

RocksDB, a project closely related to React in the Facebook ecosystem, has recently re-licensed under AL v2.0. This precedent provides a clear path and justification for React to follow suit, ensuring consistency and facilitating collaboration within the open-source community.

The growth of the React ecosystem and increased adoption in ASF projects make this re-licensing more feasible and important. Modern tools like SPDX (Software Package Data Exchange) have made managing and expressing software licenses more straightforward, further supporting this change.

## 3. Detailed Design

The proposed change involves re-licensing React.js under the Apache License v2.0. This would require:

- Updating licensing headers in all React source files.
- Modifying documentation to reflect the new licensing.
- Possibly adjusting scripts or processes for managing licenses within the React ecosystem.

To maintain compatibility with projects requiring dual licensing, React could consider a secondary license, similar to RocksDB's approach with GPL 2.0. However, the primary goal is to align with ASF standards, making AL v2.0 the focus.

No core architectural changes to React's codebase are anticipated, as this change is purely legal and administrative. Clear communication and coordination with the React community, including contributors, users, and related projects, will be crucial for a smooth transition.

## 4. Drawbacks

- Potential legal complexities and costs associated with re-licensing.
- The need for community coordination and possible re-licensing of derivative works.
- Potential pushback from contributors or users who may prefer the existing licensing.
